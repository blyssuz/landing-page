Rewrite the 1536-line TenantPage.tsx monolith into a slim orchestrator (~100-200 lines). This is the critical wiring task that connects all decomposed components.

**Structure:**
```
'use client';

// Imports: all _components, _lib modules, hooks
// (Keep server actions import for handleBookService)

export function TenantPage({ business, services, employees, photos, reviews, tenantSlug, businessId, locale, savedUser }: TenantPageProps) {
  // 1. Hooks: useDistance, useActiveSection, useRouter, usePathname
  // 2. Derived state: primaryColor, openStatus, closingTime, translations, basePath
  // 3. Section refs
  // 4. Gallery state (showGallery, currentImageIndex)
  // 5. Services state (searchQuery, activeCategory, bookingServiceId) -- KEEP AS-IS for now
  // 6. Render: sections in order

```

**Key wiring decisions:**

1. **CSS custom property:** Set `--primary` on root div: `style={{ '--primary': primaryColor } as React.CSSProperties}`.

2. **Hero section:** Conditional render based on `photos.length > 0`:
   - Has photos: render HeroMosaic (hidden lg:block) + HeroCarousel (lg:hidden). Both receive `onPhotoClick` that sets gallery index and opens lightbox. LanguageSwitcher positioned absolute top-right of hero container.
   - No photos: render HeroEmpty.
   - GalleryLightbox rendered at page level, controlled by `showGallery` state.

3. **BusinessHeader:** Pass `business`, `hasPhotos`, `locale`. The BusinessHeader internally renders MetadataBadges. Pass distance data from useDistance hook.

4. **TabNavigation:** Define tabs array from section refs. Pass `activeTab` from useActiveSection hook.

5. **Two-column layout:** `lg:grid lg:grid-cols-[1fr_380px] lg:gap-8`.
   - Left column: Services section (KEEP EXISTING INLINE -- do NOT extract. This is Phase 3 work. Copy the services rendering code (lines 914-1027) directly into the orchestrator. This keeps the file around ~200 lines instead of ~100 but avoids creating a component that Phase 3 will immediately replace.)
   - Left column continued: TeamSection (with ref), ReviewsSection (with ref).
   - Right column: DesktopSidebar (with aboutRef for IO tracking on desktop).

6. **Mobile about:** AboutSection below the two-column grid, lg:hidden, with ref for IO.

7. **aboutRef handling (CRITICAL -- see Research pitfall 5):** The IntersectionObserver needs to track the About section on both mobile and desktop. On desktop, the DesktopSidebar IS the about content. On mobile, the AboutSection IS the about content. Solution: create a single aboutRef, and conditionally attach it. Use a wrapper div that holds both (only one visible at a time via lg:hidden / hidden lg:block), and attach aboutRef to a small sentinel div that spans both contexts. OR: use two refs (desktopAboutRef, mobileAboutRef) and have useActiveSection observe both, but only the visible one triggers.

   Recommended approach: Pass aboutRef to both components. Both will render (one hidden via CSS), but IntersectionObserver only fires for the visible one (hidden elements have zero intersection). This is the simplest correct solution.

8. **Location permission modal:** Render the location permission modal (lines 1388-1444 of original) inline in the orchestrator. It uses state from useDistance hook. Use the existing motion-based bottom sheet pattern. Replace zinc-* with stone-*, remove dark:.

9. **Bottom navigation:** BottomNav component at the bottom, lg:hidden.

10. **Service booking handler:** Keep `handleBookService` function (uses `setBookingIntent` server action, router.push to booking page).

11. **switchLocale function:** Keep inline (used by LanguageSwitcher onSwitch callback).
