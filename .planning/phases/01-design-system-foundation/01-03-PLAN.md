  <name>Task 2: Create Modal component with native dialog</name>
  <files>app/components/ui/Modal.tsx</files>
  <action>
Create Modal using native `<dialog>` element following the research pattern.

**Props:**
```typescript
export interface ModalProps {
  open: boolean;
  onClose: () => void;
  children: React.ReactNode;
  className?: string;
  /** Whether clicking the backdrop closes the modal. Default: true */
  closeOnBackdrop?: boolean;
  /** Whether pressing Escape closes the modal. Default: true */
  closeOnEscape?: boolean;
}
```

**Implementation requirements:**

1. Use `<dialog>` HTML element with `showModal()` / `close()` API
2. Use `useRef` for dialog element reference (combine with forwarded ref via `useImperativeHandle` or resolved ref pattern)
3. Use `useEffect` synced to `open` prop:
   - When `open` transitions to true and dialog is not open:
     - Save `window.scrollY`
     - Set `document.body.style.position = 'fixed'`
     - Set `document.body.style.top = -${scrollY}px`
     - Set `document.body.style.width = '100%'`
     - Call `dialog.showModal()`
   - When `open` transitions to false and dialog is open:
     - Call `dialog.close()`
     - Read saved scrollY from `document.body.style.top`
     - Reset `document.body.style.position`, `top`, `width` to `''`
     - Restore scroll position with `window.scrollTo(0, parseInt(scrollY || '0') * -1)`
   - CRITICAL: Clean up on unmount -- if dialog is open when component unmounts, restore body scroll

4. Escape key handling: listen for `onCancel` event on dialog (native behavior). Call `e.preventDefault()` then `onClose()` when `closeOnEscape` is true. When `closeOnEscape` is false, just `e.preventDefault()` to suppress native close.

5. Backdrop click: listen for `onClick` on the `<dialog>` element. If `e.target === dialogRef.current` (click was on the dialog element itself, not children), and `closeOnBackdrop` is true, call `onClose()`.

6. Styling:
   - Dialog: `rounded-2xl p-0 m-auto max-w-lg w-[calc(100%-2rem)] max-h-[90vh] overflow-y-auto bg-white shadow-warm-lg`
   - Backdrop: `backdrop:bg-black/50 backdrop:backdrop-blur-[2px]`
   - On mobile (default): add `sm:max-w-lg` to constrain on larger screens
   - The dialog content wrapper should prevent click-through to dialog element: wrap `{children}` in a `<div>` that stops propagation is NOT needed -- the children are inside the dialog and won't match `e.target === dialogRef.current`

7. Animation (CSS only, not Motion -- dialog ::backdrop cannot be animated by JS):
   - Dialog open: `animate-[scaleIn_200ms_ease-out]` -- define a simple CSS keyframe if the shimmer keyframe pattern isn't sufficient. Actually, rely on the user's existing globals.css or add inline. The simplest approach: use `open:animate-[fadeIn_200ms_ease-out]` with `@starting-style` if supported, or just skip animation for restrained Airbnb feel.
   - For simplicity per the restrained animation feel: NO animation on dialog open/close. The content inside can use Motion, but the dialog shell appears/disappears cleanly.

8. Export both `Modal` and `ModalProps` as named exports. Use `'use client'` directive.

9. forwardRef to `HTMLDialogElement` with displayName.

**Testing guidance for the executor:**
- After implementing, test by temporarily importing Modal in any page and toggling `open` state
- Verify on mobile viewport: background should NOT scroll when modal is open
- Verify Escape key closes the modal
- Verify clicking the dark backdrop closes the modal
- Verify clicking inside modal content does NOT close it
