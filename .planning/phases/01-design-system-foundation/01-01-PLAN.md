**lib/animations.ts:**

Create the centralized Motion animation presets file with these exports:

1. `springs` object with three spring transition presets:
   - `gentle`: `{ type: 'spring', stiffness: 120, damping: 20 }` -- most UI transitions
   - `snappy`: `{ type: 'spring', stiffness: 300, damping: 25 }` -- buttons, chips, toggles
   - `soft`: `{ type: 'spring', stiffness: 80, damping: 18 }` -- panels, sheets, overlays

2. Animation variant objects (type `Variants` from `motion/react`):
   - `fadeUp`: hidden `{ opacity: 0, y: 16 }` -> visible `{ opacity: 1, y: 0 }` with `springs.gentle`
   - `fadeIn`: hidden `{ opacity: 0 }` -> visible `{ opacity: 1 }` with `springs.gentle`
   - `scaleIn`: hidden `{ opacity: 0, scale: 0.95 }` -> visible `{ opacity: 1, scale: 1 }` with `springs.snappy`
   - `slideUp`: hidden `{ opacity: 0, y: 24 }` -> visible `{ opacity: 1, y: 0 }` with `springs.soft`

3. Utility functions:
   - `staggerChildren(staggerMs = 0.05)`: returns `{ visible: { transition: { staggerChildren: staggerMs } } }`

4. All exports as named exports. Import `Transition` and `Variants` types from `motion/react`.

5. Add `exitVariant` for common exit animation: `{ opacity: 0, y: -8, transition: springs.snappy }`

**app/components/ui/_lib/cn.ts:**

Create a simple className merge utility -- no external dependencies. Use template literal concatenation that filters out falsy values:

```typescript
export function cn(...classes: (string | undefined | null | false)[]): string {
  return classes.filter(Boolean).join(' ');
}
```

This is intentionally simple per the research recommendation (no tailwind-merge needed for this project's component complexity).
